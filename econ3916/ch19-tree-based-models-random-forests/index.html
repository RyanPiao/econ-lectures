Gini Impurity: The Split Criterion

How "mixed" is this group?

$$G(p) = 1 - \sum_{k=1}^{K} p_k^2$$

  • \(G = 0\) → perfectly pure (all one class)
  • \(G = 0.5\) → maximally impure (50/50 mix)

Worked example: Node has 80 pos, 20 neg
\(G = 1 - (0.80^2 + 0.20^2) = 1 - 0.68 = 0.32\)

Work through the calculation on the board. Have students shout out each step.

How the Tree Chooses Splits

$$\Delta G = G(\text{parent}) - \left[\frac{n_L}{n}\, G(\text{left}) + \frac{n_R}{n}\, G(\text{right})\right]$$

The algorithm evaluates every feature and every threshold , picking the split that maximizes impurity reduction.

Greedy: Locally optimal at each step. No global lookahead.

Regression: Replace Gini with MSE. Predict \(\bar{y}_{\text{leaf}}\).

The algorithm is greedy — it makes the locally optimal choice at each step without looking ahead.

The Variance Problem

Why Single Trees Fail

Single Trees: Flexible but Unstable

Small data changes → entirely different trees.

Tree A: MedInc > 5.03 (root split)
Tree B: MedInc > 4.87 (root split)
By level 3: almost no splits in common

The solution: If B independent estimators each have variance σ², their average has variance σ²/B.

Same principle as the Central Limit Theorem.

What if we just built a hundred trees and averaged them? Pause. Let students think. This is the transition to bagging.

Bagging

Bootstrap Aggregating (Breiman, 1996)

How Bagging Works

  1. Draw B bootstrap samples (with replacement)
  2. Train one decision tree on each sample
  3. Average predictions: \(\hat{f}_{\text{bag}}(x) = \frac{1}{B}\sum_{b=1}^{B}\hat{f}_b(x)\)

OOB: Each tree misses ~37% of observations → free cross-validation

Bootstrap samples share about 63% of observations. The trees are not independent — they are correlated.

The Correlation Floor

$$\text{Var}(\bar{f}) = \underbrace{\rho\sigma^2}_{\text{FLOOR}} + \underbrace{\frac{1-\rho}{B}\sigma^2}_{\text{shrinks with B}}$$

First term \(\rho\sigma^2\): Stays no matter how many trees. This is the floor.

Second term: Shrinks toward 0 as B → ∞. This is the averaging benefit.

The question: can we reduce ρ?

This is the most important equation in the lecture. Circle the first term. No matter how many trees, you cannot push variance below this floor.

Numerical Example: Bagging

\(\sigma^2 = 0.50\), \(B = 100\) trees, \(\rho = 0.85\) (bagging)

\(\text{Var} = 0.85 \times 0.50 + \frac{0.15}{100} \times 0.50 = 0.425 + 0.001 = \mathbf{0.426}\)

The floor is 0.425. Adding more trees barely helps — we are stuck.

To go lower, we need to attack ρ.

Write this on the board. Say: 'With bagging, variance is 0.426 but the floor is 0.425. We need to attack rho.'

Common Misconception

"More trees = overfitting"

Reality: Adding trees does NOT increase overfitting. The \((1-\rho)/B\) term decreases monotonically.

RF CAN overfit through:

  • max_depth too high (trees memorize)
  • max_features too close to p (trees correlate)

n_estimators is the one hyperparameter you can safely increase.

Decision Scenario: The Feature Importance Trap

You advise a bank on mortgage default prediction. Random Forest achieves excellent AUC. Feature importance: "prior credit inquiries" is #1.

The CRO says: "Reject anyone with > 3 inquiries."

What is wrong with this reasoning?

Feature importance measures prediction, not causation. People about to default take out MORE inquiries because they're desperate for credit.
