Section 1

The Grammar of Time Series

Three components. One lag operator. Memory matters.

Every Time Series = Three Components

Trend $T_t$

Long-run direction. Is GDP growing? Is ridership declining?

Seasonality $S_t$

Repeating patterns. December retail spikes. January unemployment claims.

Residual $R_t$

Everything left. Noise + structure you missed.

$$Y_t = T_t + S_t + R_t$$

Additive decomposition — the foundation of everything that follows

Progressive build: reveal each component one at a time. Draw on board: trend line + seasonal wave + noise cloud = observed. "This visual is the conceptual anchor for the whole chapter."

Drag to rotate • Smoothing slider changes the trend

The Lag Operator $L$

$$Ly_t = y_{t-1} \qquad L^k y_t = y_{t-k}$$

$L$ simply "looks backward" one period

Differencing

$\Delta y_t = y_t - Ly_t = (1-L)y_t$

Today minus yesterday

Why it matters

In Ch 21, $L$ becomes the backbone of ARIMA: $\phi(L)y_t = \theta(L)\varepsilon_t$

Write L on the board. Physically point backward. "L literally means look backward one step." Keep this brief — 2 min.

Key Concept: Temporal Dependence

Time series observations are not independent . Each value depends on its predecessors through $L$. This invalidates:

  • Random shuffling in cross-validation (Ch 15)
  • Standard bootstrap resampling (Ch 7)
  • i.i.d. error assumptions of OLS (Ch 12)

From today forward: every method must respect chronological order.

This is the bridge from Ch 12-19 to Ch 20+. Emphasize the break: "Everything you learned still works. But the rules of APPLICATION change."

Class Poll

A retail chain's December revenue is always 20% higher than November. In 2024, December was 35% higher . Which component explains the extra 15%?

A

Trend — the economy is growing faster

B

Seasonality — December is always high

C

Residual — something unusual happened

D

Structural break — pattern permanently changed

Ask for show of hands. D is the tempting wrong answer.

Answer: C — Residual

The 20% regular December bump is seasonality . The extra 15% above that pattern is captured in the residual — it could be a one-time promotional success, a competitor closing, or post-election spending.

Why not D (structural break)? One data point isn't enough. If the 35% pattern repeats in future Decembers, then we might reclassify it. But a single surprising observation is a residual, not a permanent structural change.

Section 2

Decomposition

Taking the series apart to find the surprise

STL Decomposition

STL (Cleveland et al. 1990) uses LOESS regression to estimate trend and seasonal components. Three advantages over classical:

  • Trend can be nonlinear
  • Seasonal pattern can evolve over time
  • robust=True downweights outliers

Additive vs. Multiplicative: If seasonal swings grow with the level → multiplicative (or log + additive)

Section 3

Stationarity

The foundation of forecasting

What Is Stationarity?

A stationary series has stable statistical properties over time:

  1. $E[Y_t] = \mu$ — constant mean
  2. $\text{Var}(Y_t) = \sigma^2$ — constant variance
  3. $\text{Cov}(Y_t, Y_{t-k}) = \gamma(k)$ — depends only on lag $k$

Why it matters: Models learn patterns from the past. If patterns change, the model learns rules that expire .

"If you estimate 2.5% growth from 2010-2019 data, you assume the process is stable. If a break occurred in 2020, your estimate describes a world that no longer exists."

The Random Walk & Differencing

Random Walk (Non-Stationary)

$Y_t = Y_{t-1} + \varepsilon_t$

  • No tendency to revert to any level
  • Variance grows: $\text{Var}(Y_t) = t\sigma^2$
  • Wanders aimlessly

After Differencing (Stationary)

$\Delta Y_t = \varepsilon_t$

  • White noise: zero mean, constant variance
  • If $d$ differences needed: series is $I(d)$
  • GDP levels: $I(1)$. GDP growth: $I(0)$.
Show GDP levels vs differences on board. "The visual contrast is immediate: one trends up, the other fluctuates around zero."

Testing GDP Stationarity 

from statsmodels.tsa.stattools import adfuller, kpss

# GDP Levels
adf_p = adfuller(gdp.dropna(), autolag='AIC')[1]
kpss_p = kpss(gdp.dropna(), regression='c')[1]
# ADF p=0.99, KPSS p=0.01 → NON-STATIONARY

# GDP First Differences
adf_p = adfuller(gdp.diff().dropna())[1]
kpss_p = kpss(gdp.diff().dropna())[1]
# ADF p=0.0001, KPSS p=0.10 → STATIONARY ✓

Levels: ADF fail, KPSS reject → Non-Stationary

Differences: ADF reject, KPSS fail → Stationary ✓

Run this live. Show that GDP levels fail the test, differences pass. "GDP is I(1) — its first difference is the modeling target."

Class Poll

You test monthly CPI: ADF fails to reject (p=0.42). KPSS rejects (p=0.01). Diagnosis?

A

Stationary — safe to model

B

Non-stationary — difference it

C

Trend-stationary — detrend it

D

Inconclusive — tests contradict

Ask for show of hands. D is the tempting wrong answer.

Answer: B — Non-stationary

ADF's null is "unit root" — failing to reject means we can't rule out non-stationarity. KPSS's null is "stationary" — rejecting means we have evidence against stationarity. Both tests point the same direction.

Why not D (inconclusive)? Students confuse "opposite null hypotheses" with "contradictory results." The tests have opposite nulls precisely so they can confirm each other. Here they AGREE: the series is non-stationary. Take first differences and re-test.

Section 4

Structural Breaks

When the rules of the game change permanently

Types of Structural Breaks

Level Shift

MBTA ridership: 600K → 50K overnight, recovered to only 400K (permanent -30%)

Trend Break

Nvidia: linear growth → exponential after ChatGPT. Growth rate permanently changed.

Variance Break

Stock volatility: 15% → 80%+ during 2008 and 2020 crises.

Parameter Break

Phillips Curve: unemployment 3.4% + inflation >3% contradicted all pre-COVID models.

Go through each type quickly. "All four share one property: models trained on pre-break data are WRONG after the break."

Detecting Breaks with ruptures 

import ruptures as rpt
import numpy as np

# Simulated Nvidia-like revenue
pre  = np.random.normal(3.5, 0.5, 20)   # ~$3.5B
post = np.random.normal(15.0, 3.0, 12)  # ~$15B
data = np.concatenate([pre, post])

# PELT algorithm: detect mean shifts
algo = rpt.Pelt(model="l2").fit(data)
breakpoints = algo.predict(pen=10)
# → [20, 32]  ← break at quarter 20 ✓

pen parameter: Higher penalty = fewer, more confident breaks. Lower penalty = more sensitive detection. Always test multiple values.

Run live. Show the break detection. "The algorithm found the break at exactly quarter 20 — where we planted it."

Time Series Cross-Validation

The rule from today forward: never shuffle time series data.

  • Expanding window: Training set grows with each fold
  • Sliding window: Fixed-length training window moves forward
  • Use sliding when old data describes a different regime

TimeSeriesSplit replaces KFold . Always.
