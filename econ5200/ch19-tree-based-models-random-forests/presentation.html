Gini Impurity: The Split Criterion

$$G(p) = 1 - \sum_{k=1}^{K} p_k^2$$

G = 0

Pure node
(all one class)

G = 0.32

Moderate impurity
(80/20 split)

G = 0.50

Maximum impurity
(50/50 split)

Split criterion: $\Delta G = G(\text{parent}) - \left[\frac{n_L}{n}G(\text{left}) + \frac{n_R}{n}G(\text{right})\right]$

The algorithm is greedy — picks the locally best split at each step. No global lookahead.

Write Gini formula on board. "This measures how mixed a node is." Do the 80/20 calculation live.

Worked Example: Gini Calculation

A node contains 80 positives and 20 negatives (n = 100):

$p_{+} = 0.80, \quad p_{-} = 0.20$

$G = 1 - (0.80^2 + 0.20^2) = 1 - (0.64 + 0.04) = \mathbf{0.32}$

Now split into Left (75+, 0−) and Right (5+, 20−):

Left: G = 0 (pure!)

Right: G = 1 − (0.20² + 0.80²) = 0.32

Weighted: $\frac{75}{100}(0) + \frac{25}{100}(0.32) = 0.08$ — impurity dropped from 0.32 to 0.08

Common Misconception

"Decision trees are too simple to overfit."

The opposite is true. An unrestricted tree achieves zero training error by splitting until each leaf has one observation. A tree with 1,000 leaves is not a simple model.

Tree hyperparameters ( max_depth , min_samples_leaf ) work like the regularization penalty λ from Chapter 16 — they limit structural complexity.

An unrestricted tree memorizes training data, losing 30-40 percentage points on test data. This is the high-variance extreme of the bias-variance tradeoff.

Common mistake: Assuming that simple splits → simple model. But 1,000 splits = 1,000 leaves = a very complex model.

Key insight: The solution is not to restrict the tree (re-introduces bias). Instead: build many trees and average them.

Common Misconception

"More trees → the forest will eventually overfit."

Adding trees does NOT increase overfitting. The (1−ρ)/B term decreases monotonically. n_estimators is a "free" hyperparameter — set it to 500 and forget it.

RF overfits through other mechanisms: max_depth too high, max_features too close to p.

Class Poll

B

With B=500, the 1/B term is negligible. The floor ρσ² dominates. Only reducing ρ (via max_features) helps.

Answer: B — Reduce max_features

With B=500, the (1−ρ)/B term is already negligible. The dominant term is the correlation floor ρσ² . Adding more trees barely touches it.

The most important RF hyperparameter is max_features, not n_estimators.

n_estimators is "free" — set and forget. max_features controls the bias-variance tradeoff within the ensemble.

Interpretation & Causation

Feature importance tells you what predicts — not what causes

Feature Importance (MDI)

For each feature j, sum impurity reductions across all splits across all trees.

Caveat: MDI inflates importance of high-cardinality continuous features. Permutation importance is less biased.

Prediction ≠ Causation

Feature Importance Says

"MedInc predicts housing prices"

Correlation. Confounding possible.

Does NOT Mean

"Raising income causes higher prices"

Reverse causality. Proxy effects.

Key Takeaways

Decision trees partition feature space via Gini impurity — flexible but overfit severely.

Bagging averages trees to reduce variance, but hits the correlation floor ρσ².

Random Forests break the floor via feature randomization — diverse errors cancel when averaged.

Prediction ≠ Causation. Feature importance measures what predicts, not what causes.
