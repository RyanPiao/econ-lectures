Decision Trees: 20 Questions for Data

A tree asks binary questions about features:

  • "Is income > $50,000?"
  • "Is the property downtown?"
  • "Are reviews > 4.5 stars?"

At each split, it creates groups as homogeneous as possible.

Key Insight for Economists

Unlike OLS (one straight line), a tree learns that income's effect differs below $30K vs above $80K — no hand-engineered interactions required.

Instead of fitting a global function, the tree partitions feature space into rectangles and fits a constant in each.

Draw a 2D scatter: income vs price. Draw OLS line. Then draw two vertical splits creating three rectangles. "OLS gives one slope. The tree gives three predictions."

Gini Impurity: The Split Criterion

$$G(p) = 1 - \sum_{k=1}^{K} p_k^2$$

G = 0

Pure node
(all one class)

G = 0.32

Moderate impurity
(80/20 split)

G = 0.50

Maximum impurity
(50/50 split)

Split criterion: $\Delta G = G(\text{parent}) - \left[\frac{n_L}{n}G(\text{left}) + \frac{n_R}{n}G(\text{right})\right]$

The algorithm is greedy — picks the locally best split at each step. No global lookahead.

Write Gini formula on board. "This measures how mixed a node is." Do the 80/20 calculation live.

Worked Example: Gini Calculation

A node contains 80 positives and 20 negatives (n = 100):

$p_{+} = 0.80, \quad p_{-} = 0.20$

$G = 1 - (0.80^2 + 0.20^2) = 1 - (0.64 + 0.04) = \mathbf{0.32}$

Now split into Left (75+, 0−) and Right (5+, 20−):

Left: G = 0 (pure!)

Right: G = 1 − (0.20² + 0.80²) = 0.32

Weighted: $\frac{75}{100}(0) + \frac{25}{100}(0.32) = 0.08$ — impurity dropped from 0.32 to 0.08

Common Misconception

"Decision trees are too simple to overfit."

The opposite is true. An unrestricted tree achieves zero training error by splitting until each leaf has one observation. A tree with 1,000 leaves is not a simple model.

Tree hyperparameters (max_depth, min_samples_leaf) work like the regularization penalty λ from Chapter 16 — they limit structural complexity.

Common mistake: Assuming that simple splits → simple model. But 1,000 splits = 1,000 leaves = a very complex model.

Key insight: The solution is not to restrict the tree (re-introduces bias). Instead: build many trees and average them.

The Averaging Solution

From the CLT to Bootstrap Aggregating

The Variance Problem

Small changes in training data → entirely different trees.

Two trees on random 80/20 splits of California Housing:

  • Tree 1: root split at MedInc > 5.03
  • Tree 2: root split at MedInc > 4.87

By level 3, they share almost no splits.

The Statistical Fix

If B independent estimators each have variance σ², their average has variance:

$$\frac{\sigma^2}{B}$$

Same principle as the CLT. Averaging reduces variance.

"We have a flexible but unstable model. And we know averaging reduces variance. What if we just built a hundred trees and averaged them?"

Bagging: Bootstrap Aggregating

1

Draw B bootstrap samples (random with replacement) from training data

2

Train one decision tree on each bootstrap sample

3

Average predictions: $\hat{f}_{\text{bag}}(x) = \frac{1}{B}\sum_{b=1}^{B} \hat{f}_b(x)$

~63% of data in each sample

~37% "out-of-bag" → free validation

The Correlation Floor

Bootstrap samples share ~63% of data → trees are correlated.

$$\text{Var}(\bar{X}) = \underbrace{\rho\sigma^2}_{\text{correlation floor}} + \underbrace{\frac{1 - \rho}{B}\sigma^2}_{\to 0 \text{ as } B \to \infty}$$

As B → ∞: Second term vanishes

Floor ρσ² remains — can't average past it

To break the floor: Reduce ρ

Force trees to be different → Random Forests

Build step-by-step on board. Step 1: independent case (ρ=0), Var=σ²/B. Step 2: add correlation. Step 3: as B→∞, only ρσ² remains.

Bagging averages trees that share ~63% of their data. Why is this shared data a problem?

Most will identify shared data makes trees similar. Guide toward: "What if each tree could only see a random subset of features at each split?"

The Fix: Forced Diversity

Instead of letting every tree see all features at every split...

Randomly restrict each split to a subset of features.

Some trees are forced to split on AveRooms or Latitude first — even when MedInc is globally best.

Different features → different errors → better averaging.

Breaking the Correlation Floor

Random Forests: One Twist That Changes Everything

Random Forests (Breiman, 2001)

Bagging + one twist: at each split, only m randomly selected features are considered.

Classification

$m = \sqrt{p}$

p=100 → m=10

Regression

$m = p/3$

p=100 → m≈33

Why it works: If every tree splits on MedInc first → all trees make similar errors (high ρ). Feature restriction forces diversity → different errors → better cancellation when averaged.

The Numbers: 52% Lower Variance

Individual tree variance σ² = 0.50, B = 100 trees:

Bagging (ρ = 0.85)

Var = 0.85(0.50) + 0.15/100(0.50)

= 0.426

Random Forest (ρ = 0.40)

Var = 0.40(0.50) + 0.60/100(0.50)

= 0.203

Feature randomization reduced ρ from 0.85 → 0.40, cutting the floor in half.

Common Misconception

"More trees → the forest will eventually overfit."

Adding trees does NOT increase overfitting. The (1−ρ)/B term decreases monotonically. n_estimators is a "free" hyperparameter — set it to 500 and forget it.

RF overfits through other mechanisms: max_depth too high, max_features too close to p.

Class Poll

RF with B=500 trees, ρ=0.6. Which action reduces variance MORE?

A

Add 500 more trees (B=1000)

B

Reduce max_features from p to √p

C

Both reduce variance equally

D

Neither — 500 trees is enough

With B=500, the 1/B term is negligible. The floor ρσ² dominates. Only reducing ρ (via max_features) helps.

Answer: B — Reduce max_features

With B=500, the (1−ρ)/B term is already negligible. The dominant term is the correlation floor ρσ². Adding more trees barely touches it.

The most important RF hyperparameter is max_features, not n_estimators.

n_estimators is "free" — set and forget. max_features controls the bias-variance tradeoff within the ensemble.

Interpretation & Causation

Feature importance tells you what predicts — not what causes

Feature Importance (MDI)

For each feature j, sum impurity reductions across all splits across all trees.

Caveat: MDI inflates importance of high-cardinality continuous features. Permutation importance is less biased.

Select observations • Red = pushes prediction up • Blue = pushes prediction down

Prediction ≠ Causation

Feature Importance Says

"MedInc predicts housing prices"

Correlation. Confounding possible.

Does NOT Mean

"Raising income causes higher prices"

Reverse causality. Proxy effects.

Key Takeaways

Decision trees partition feature space via Gini impurity — flexible but overfit severely.

Bagging averages trees to reduce variance, but hits the correlation floor ρσ².

Random Forests break the floor via feature randomization — diverse errors cancel when averaged.

Prediction ≠ Causation. Feature importance measures what predicts, not what causes.
